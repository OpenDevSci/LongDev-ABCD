Getting Started üöÄ

Before diving into contributions, please familiarize yourself with our project‚Äôs README. It provides an essential overview of LongDev-ABCD, setting the stage for meaningful contributions.

Internal Members

Repository Members: 1. Clone the Repository Directly

To clone the repository to your local machine, use the following command in your terminal. Replace repository-url with the actual URL of the repository you want to clone.

git clone https://github.com/your-organization/repository-url.git
cd repository-url

  1. Create a New Branch for Changes

    Before making any changes, it‚Äôs best practice to create a new branch. This keeps your modifications organized and separate from the main branch. Replace new-branch-name with a descriptive name for your branch.

    git checkout -b new-branch-name

  2. Push the Branch to the Repository

    After committing your changes to the new branch, push the branch to the GitHub repository. This makes your branch available to others for review and further contributions.

    git push origin new-branch-name

  3. Open a Pull Request for Review

    Once your branch is pushed to GitHub, navigate to the repository page on GitHub. You‚Äôll see a prompt to ‚ÄúCompare & pull request‚Äù for your recently pushed branches. Click it to open a pull request.

    • Provide a title and a detailed description of your changes.
    • Mention any specific collaborators for review if necessary.
    • Submit the pull request by clicking the ‚ÄúCreate pull request‚Äù button.

Remember, pull requests are a great way to get feedback and make your contributions more effective.

External Contributors

For those not a repository member: External contributors should start by forking the repository. This creates a copy of the repository in your own GitHub account, allowing you to make changes without affecting the main project until your contributions are reviewed and accepted.

  1. Fork the Repository: Visit the LongDev-ABCD repository on GitHub and click the ‚ÄúFork‚Äù button at the top right corner.
  2. Clone Your Fork: Clone the forked repository to your local machine.
# Clone your fork of the repository
git clone https://github.com/yourusername/LongDev-ABCD.git
# Navigate into the repository directory
cd LongDev-ABCD

Replace yourusername with your GitHub username.

  1. Create a new branch for your changes.
  2. Push changes to your fork.
  3. Open a pull request against the main LongDev-ABCD repository.

Creating and Using Branches

Regardless of being an internal member or an external contributor, the next step is to create a new branch for your work:

# Create a new branch for your changes
git checkout -b feature-branch-name

Replace feature-branch-name with a descriptive name for your branch.

This branch is where you‚Äôll make changes, commit updates, and push modifications back to the repository (or your fork) before opening a Pull Request.

For more details on making contributions, including committing changes and opening Pull Requests, refer back to the Understanding GitHub Flow section. ## Understanding GitHub Flow üîÑ

GitHub Flow is a lightweight, branch-based workflow that supports teams and projects where deployments are made regularly. Here‚Äôs how we apply GitHub Flow in LongDev-ABCD:

  1. Create a branch: For any new work, create a branch off the main branch. Branch names should be descriptive and reflect the task at hand.
  2. Add commits: Make changes to your branch and commit them. Commit messages should clearly explain what has been changed and why.
  3. Open a Pull Request (PR): Push your branch to GitHub and open a Pull Request. Provide a summary of your work and any additional context needed for reviewers.
  4. Discuss and review: The team will review your changes. Engage in discussions and make any required updates based on feedback.
  5. Deploy: Once your PR is approved, it will be merged into the main branch and deployed to ensure it works in production.
  6. Merge: After deployment, your changes will be merged into the main branch.
